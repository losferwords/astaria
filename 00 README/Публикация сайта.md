# Публикация сайта

Публичная версия Астарии собирается через Quartz из заметок, где явно указано:

```yaml
publish: true
```

Черновики, служебные папки, WorldAnvil backup, `.ai/`, `06 Черновики/` и игровые материалы не попадают в публикацию по умолчанию.

## Локальная проверка

```bash
ruby _scripts/sync_quartz_content.rb
cd _quartz
npm ci
NODE_OPTIONS=--max-old-space-size=8192 node quartz/bootstrap-cli.mjs plugin install --from-config
NODE_OPTIONS=--max-old-space-size=8192 node quartz/bootstrap-cli.mjs build -d content -o public
NODE_OPTIONS=--max-old-space-size=8192 node quartz/bootstrap-cli.mjs build -d content -o public --serve --port 8080
```

Локальный сайт открывается по адресу:

```text
http://localhost:8080
```

## GitHub Pages

После коммита и `git push` workflow `.github/workflows/deploy-quartz.yml` соберёт сайт автоматически.

В репозитории GitHub нужно открыть:

```text
Settings -> Pages -> Build and deployment -> Source
```

И выбрать:

```text
GitHub Actions
```

После успешного workflow сайт будет доступен по адресу:

```text
https://losferwords.github.io/astaria/
```

## Карты

Оригинальные карты остаются в `Assets/Maps/`.

Для сайта используются облегчённые web-копии:

- `Assets/Maps/Web/states-web.jpg`
- `Assets/Maps/Web/heightmap-web.jpg`
- `Assets/Maps/Web/biomes-web.jpg`

Публикационный скрипт сам заменяет ссылки на карты только в Quartz-копии, не меняя исходные заметки Obsidian.
